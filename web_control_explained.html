What Is This Page?
📡

A Web Page Served Directly by Your ESP8266

①
No Internet Needed
The ESP8266 itself hosts the web page on your local Wi-Fi network. No cloud, no external server, no internet required. As long as your phone/PC is on the same Wi-Fi, it works.
②
No App to Install
Open any browser — Chrome, Safari, Firefox — type the IP address and the dashboard loads instantly. Works on phone, tablet, and PC.
③
Stored in ESP8266 Flash Memory
The entire HTML/CSS/JavaScript of this page is stored inside the ESP8266's flash using PROGMEM. Every time you load it, the microcontroller sends it to your browser.
Every UI Element — Explained
🧩

What Each Part of the Dashboard Does

1
🌀 Spinning Fan Icon
The large circle at the top. When the fan is ON it glows green and spins continuously using CSS animation. When OFF it turns red and stays still. Gives you an instant visual status at a glance — no reading needed.
2
Status Badge (FAN ON / FAN OFF)
The pill-shaped badge below the fan ring. Shows FAN ON with a green pulsing dot when running, or FAN OFF in grey when not. Updates automatically — you never need to refresh the page.
3
⚙ Servo 1 Card (ON Motor)
Left card showing status of Servo 1 on pin D1 · GPIO5. Shows Standby normally. When you press Turn ON, it lights up green and shows PRESSING... for the duration of the servo movement, then returns to Standby.
4
⚙ Servo 2 Card (OFF Motor)
Right card showing status of Servo 2 on pin D2 · GPIO4. When you press Turn OFF, it lights up red and shows PRESSING.... Each servo card activates only for its own command — they never both activate at once.
5
✓ Turn ON Button
Sends command /on to ESP8266. Only Servo 1 fires — it presses the switch to the ON position. The button automatically greys out and becomes disabled when the fan is already ON, preventing a double command.
6
✕ Turn OFF Button
Sends command /off to ESP8266. Only Servo 2 fires — it presses the switch to the OFF position. Greys out when fan is already OFF. The hint text below each button shows which servo fires and which pin it uses.
7
Info Strip (Bottom Bar)
Shows three pieces of live data: IP ADDRESS of the ESP8266, LAST SYNC timestamp of the last status poll, and COMMANDS counter — total commands sent this session.
8
Busy Overlay (Spinner)
A semi-transparent blur overlay that covers the entire page while the servo is physically moving. Shows which servo is firing (e.g. "SERVO 1 PRESSING ON..."). All buttons are disabled during this time — prevents sending two commands while one is in progress.
9
Toast Notification
A small pill popup at the bottom of the screen that slides up, stays for 2.8 seconds, then fades away. Confirms every action: "Fan ON — Servo 1 fired" or shows "Connection error" if the ESP8266 is unreachable. Never interrupts your workflow.
10
🔧 Credit Section (Ilham Farooque)
Developer credit card at the bottom with avatar, name, and two social links. GitHub link opens github.com/ilham786 and Instagram opens your profile. Both open in a new tab so the dashboard stays open.
How It Works — Step by Step
🔁

Complete Communication Flow — What Happens When You Press a Button

01
👆

You Tap "Turn ON"

Browser JavaScript runs cmd('on'). Busy lock is set. Busy overlay appears. Servo 1 card lights up green immediately.

02
📤

HTTP Request Sent

Browser calls fetch('/on') — sends an HTTP GET request over your local Wi-Fi to http://192.168.1.105/on.

03
📡

ESP8266 Receives It

The web server running on port 80 receives the request. The handleON() function is called inside the Arduino code.

04
⚙️

Servo 1 Fires on D1

PWM signal sent to pin D1. Servo 1 arm swings from 90° (neutral) to 55° (press angle), physically pushing the switch to ON.

05
⏱

Hold for 450ms

Servo stays pressed for 450ms — enough time to ensure the physical switch clicks and latches into the ON position reliably.

06
↩️

Servo Returns to Neutral

Servo arm moves back to 90° (neutral). It is no longer touching the switch. The switch stays ON mechanically on its own.

07
📥

ESP8266 Replies JSON

ESP8266 sends back {"fan_on":true,"busy":false}. This tells the browser the confirmed new state of the system.

08
🎨

Dashboard Updates

Fan icon spins green, badge shows FAN ON, ON button greys out, toast pops "Fan ON — Servo 1 fired", command counter increments.

JavaScript Inside the Page — Explained
🧠

How the Browser Side Code Works

1. SENDING A COMMAND
function cmd(c) {
  // Ignore tap if already busy
  if (busy) {
    toast('Please wait...');
    return;
  }
  busy = true;
  fetch('/' + c)
    .then(r => r.json())
    .then(d => {
      fanOn = d.fan_on;
      draw(null);
      toast('Done!');
    })
    .finally(() => { busy = false; });
}
2. AUTO-POLLING EVERY 3 SECONDS
function poll() {
  // Silent background check
  fetch('/status')
    .then(r => r.json())
    .then(d => {
      fanOn = d.fan_on;
      draw(null);
    })
    .catch(() => {}); // silent fail
}

// Start on page load, repeat every 3s
poll();
setInterval(poll, 3000);
3. UPDATING THE UI (draw function)
function draw(activeServo) {
  if (fanOn) {
    ring.className = 'db-ring on';
    badge.className = 'db-badge on';
    btnOn.disabled = true;
    btnOff.disabled = false;
  } else {
    ring.className = 'db-ring off';
    // ... reverse
  }
  // Light up correct servo card
  if (activeServo === 'on')
    chip1.classList.add('fon');
}
4. TOAST NOTIFICATION
function toast(msg) {
  const t =
    getElementById('toast');

  t.textContent = msg;

  // Slide up + show
  t.classList.add('show');

  // Auto-hide after 2.8s
  setTimeout(() => {
    t.classList.remove('show');
  }, 2800);
}
All JavaScript uses the browser's native fetch() API — no external libraries, no jQuery. Works on every modern browser including mobile Safari and Chrome.
Dashboard States — All 4 Modes
🎨

How the Dashboard Changes Appearance in Each State

FAN ON State

  • Fan ring glows green and spins
  • Badge shows FAN ON in green
  • Turn ON button is grey / disabled
  • Turn OFF button is active red
  • Servo 1 card returns to Standby

FAN OFF State

  • Fan ring glows red, stops spinning
  • Badge shows FAN OFF in grey
  • Turn ON button is active green
  • Turn OFF button is grey / disabled
  • Servo 2 card returns to Standby

BUSY State (servo moving)

  • Blur overlay covers entire page
  • Spinner animation shown
  • Text shows which servo is firing
  • All buttons are fully disabled
  • Lasts ~650ms total (press + return)

ERROR State

  • Toast shows "Connection error"
  • Happens if ESP8266 is offline
  • Dashboard stays on last known state
  • Auto-poll resumes every 3 seconds
  • Recovers automatically when ESP8266 comes back online
How to Use It — Step by Step
📱

Accessing and Using the Dashboard From Any Device

1

Upload Code to ESP8266

Enter your Wi-Fi name and password in the code, upload to NodeMCU via Arduino IDE. Both servos will move to neutral position on startup.

2

Get IP from Serial Monitor

Open Serial Monitor at 115200 baud. After connecting you'll see: http://192.168.1.X/ — copy that address.

3

Connect Phone to Same Wi-Fi

Your phone and ESP8266 must be on the same Wi-Fi network. Turn off mobile data to avoid confusion with routing.

4

Open Any Browser

Type the IP address directly: http://192.168.1.105. The dashboard loads instantly. No login, no setup.

5

Add to Phone Home Screen

iPhone: Share → Add to Home Screen
Android: ⋮ menu → Add to Home Screen
It now opens like a native app from an icon.

6

Share With Family

Anyone on the same Wi-Fi can open the same IP address on their own phone or PC and control the fan independently. No limit on devices.

7

Control From Any Automation

Use /on and /off from Python, curl, or Home Assistant. Example: curl http://192.168.1.105/on in a terminal turns the fan on.

8

Set a Static IP (Recommended)

Add WiFi.config(ip, gateway, subnet) before WiFi.begin() in setup. Then the IP never changes after router restarts — your bookmark always works.

Auto-Sync: The dashboard quietly polls /status every 3 seconds. If someone turns the fan on from another device, your page updates automatically — no manual refresh ever needed.
